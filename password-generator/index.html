🔑 What Is a Password Generator?

A password generator is a tool that creates random, unpredictable character strings to be used as secure passwords. Unlike human-chosen passwords (which tend to follow patterns), machine-generated passwords are truly random and much harder for attackers to crack.

How Does It Work?

This tool uses the browser's built-in crypto.getRandomValues() API — a cryptographically secure random number generator. This means the randomness is suitable for security applications, not just simulations.

How to Use This Tool

  • Set a length — Longer passwords are always more secure. 16+ characters is recommended for important accounts.
  • Choose character types — Toggle uppercase, lowercase, numbers, and symbols to match the target site's requirements.
  • Exclude ambiguous characters — Enable this if you need to type the password manually. Removes 0/O, l/1, I characters.
  • Click Generate — A new random password appears instantly.
  • Copy and store — Use a password manager (like Bitwarden, 1Password, or LastPass) to save the password securely.

Password Strength Guide

🔴 Weak (< 8 chars)Easily brute-forced. Never use for real accounts.
🟡 Fair (8–11 chars)Passable for low-value accounts. Add symbols.
🟢 Strong (12–15 chars)Good for most uses. Use mixed character sets.
💪 Very Strong (16+)Excellent. Resistant to modern brute-force attacks.

Best Practices

  • Use a unique password for every website — never reuse passwords.
  • Never share passwords via email, SMS, or chat.
  • Enable two-factor authentication (2FA) wherever possible.
  • Store passwords in a dedicated password manager — never in a browser's default autofill without a master password.
  • Rotate passwords for critical accounts (email, banking) periodically.
  • Avoid password "patterns" like substituting letters with numbers (p@ssw0rd is easily guessed by modern crackers).

Why Not Use Simple Passwords?

Simple or common passwords are vulnerable to dictionary attacks (trying known words), brute force attacks (trying all combinations), and credential stuffing (using leaked password databases from other site breaches). A 6-character lowercase password has only 26⁶ = ~308 million combinations — crackable in under a second by modern hardware.

A 16-character password with all character types has over 6.2 × 10²⁹ combinations — computationally infeasible to crack by any current technology.
